# AI Design Mentor – Product Idea and Detailed Phased Plan

## 1. Product Title

**AI Design Mentor**

An AI-powered design review and improvement platform that gives professional, structured, educational, and honest feedback on visual designs.

---

## 2. Core Problem

Today, almost anyone can create designs using design tools, templates, and AI tools. However, most people do not receive meaningful feedback on their work.

Common feedback is often vague and unhelpful, such as:

- “Looks good.”
- “Something feels off.”
- “I don’t like the font.”
- “Make it more modern.”
- “The colors are not working.”

This kind of feedback does not help the user understand:

- What exactly is wrong.
- Why it is wrong.
- How to improve it.
- Which design principle is being violated.
- How the design affects the target audience.
- Whether the design fits its goal or industry.

Designers, students, freelancers, business owners, creators, and non-designers need a professional mentor-like system that can review their work clearly and practically.

---

## 3. Core Solution

AI Design Mentor is an AI-powered design mentor that reviews any type of visual design and gives professional, detailed feedback similar to an experienced designer, art director, creative director, or UI/UX mentor.

The platform does not simply say whether a design is good or bad. It explains:

- What works.
- What does not work.
- Why something feels wrong.
- Which design principle is involved.
- How to fix the issue.
- What specific changes should be made.
- How the design can better match its target audience and purpose.

The goal is not only to improve one design, but also to help the user become better at design over time.

## 7. Core User Flow

### Step 1: Upload the Design

The user uploads their visual design.

Supported upload formats can include:

- PNG
- JPG
- JPEG
- WEBP
- PDF
- SVG
- Figma screenshot exports
- Multi-image uploads for branding projects or UI flows

The first version can start with simple image upload support and later expand to PDFs, SVGs, and multi-screen projects.

---

### Step 2: Select the Design Category

The user selects the category of the uploaded design.

Example categories:

- Logo
- UI/UX screen
- Website design
- Mobile app screen
- Poster
- Book cover
- Packaging
- Branding project
- Social media post
- Illustration
- Advertisement
- Presentation slide
- Other

This helps the AI apply the correct review standards.

---

### Step 3: Provide a Design Brief

The user provides context about the design.

Required or optional fields:

- Target audience
- Purpose
- Style
- Goals
- Brand personality
- Industry
- Platform
- Design format
- Important message
- Competitor or inspiration reference
- Any specific concern

Example design brief fields:

```text
Target audience: Young professionals aged 20–35
Purpose: Promote a productivity app
Style: Minimal, modern, clean
Goal: Increase signups
Concern: Not sure if the hero section looks premium enough
```

The design brief is important because good design cannot be judged only by appearance. It must be judged according to purpose, audience, and context.

---

### Step 4: Choose Feedback Mode

The user chooses how the AI should respond.

Available feedback modes:

1. Beginner Mode
2. Mentor Mode
3. Brutally Honest Mode

Each mode should review the same design, but the language, depth, and tone should change.

---

### Step 5: Receive Detailed AI Feedback

The AI reviews the design and provides structured feedback.

The feedback should include:

- Overall score
- Category-wise scores
- What is working
- What is not working
- Why each issue matters
- Specific improvement recommendations
- Priority level for each fix
- Suggested next steps

---

### Step 6: Ask Follow-Up Questions

The user can ask follow-up questions such as:

- “Which font should I use instead?”
- “How can I make this more premium?”
- “Can you explain hierarchy in this design?”
- “Which color palette would work better?”
- “Is this suitable for my target audience?”
- “How do I improve this for Instagram?”
- “What should I change first?”

This turns the product from a one-time AI checker into an interactive design mentor.

---

### Step 7: Optionally Generate an Improved Version

The user may choose to:

- Receive suggestions only.
- Generate an improved version.
- Compare before and after versions.
- Generate alternate design directions.
- Create a revised prompt for another AI design tool.
- Export the improvement plan.

This feature should be optional because some users may want to manually improve their own design instead of letting AI redesign it.

## 8. Feedback Modes

## 8.1 Beginner Mode

### Purpose

Beginner Mode gives friendly, simple, and educational feedback.

It should be useful for people who are new to design and may not understand professional design terms.

### Tone

- Friendly
- Encouraging
- Simple
- Educational
- Patient
- Non-intimidating

### Feedback Style

Beginner Mode should explain design principles in simple language.

Example:

```text
The title is hard to notice because it is too close in size to the body text. In design, important things should look more important. You can fix this by making the title bigger, bolder, or placing it higher in the layout.
```

### Best For

- Beginners
- Students
- Non-designers
- Small business owners
- Content creators learning design

---

## 8.2 Mentor Mode

### Purpose

Mentor Mode gives professional, detailed, and practical feedback.

It should feel like an experienced designer reviewing the work and explaining improvement opportunities.

### Tone

- Professional
- Helpful
- Structured
- Detailed
- Balanced
- Skill-building

### Feedback Style

Mentor Mode should include principle-based critique, reasoning, and actionable fixes.

Example:

```text
The visual hierarchy is weak because the headline, supporting text, and CTA all compete for attention. The design needs a clearer priority structure. Increase headline contrast, reduce secondary text weight, and give the CTA more spacing so the user's eye moves naturally from message to action.
```

### Best For

- Freelancers
- Design students
- UI/UX designers
- Portfolio builders
- Founders reviewing brand assets

---

## 8.3 Brutally Honest Mode

### Purpose

Brutally Honest Mode gives direct art-director style critique.

It should be honest, clear, and sharp, but not insulting. The goal is improvement, not humiliation.

### Tone

- Direct
- Honest
- Critical
- Professional
- No unnecessary sugarcoating
- Still respectful

### Feedback Style

The feedback should clearly identify weak parts of the design and explain what must change.

Example:

```text
The design currently looks unfinished. The typography has no clear hierarchy, the spacing feels random, and the color choices do not support the serious tone mentioned in the brief. Before adding more decoration, fix the structure, spacing, and type scale.
```

### Best For

- Users who want serious critique
- Freelancers before client delivery
- Portfolio improvement
- Advanced designers
- Users who do not want vague praise

## 10. Detailed Review Category Definitions

## 10.1 Composition

Composition checks how visual elements are arranged.

The AI should review:

- Balance
- Alignment
- Layout structure
- Focal point
- Visual flow
- Use of negative space
- Element grouping
- Overall visual stability

Common issues:

- Design feels cluttered.
- Elements are randomly placed.
- No clear focal point.
- Too many competing elements.
- Layout feels unbalanced.

Recommendations may include:

- Use a grid.
- Align elements consistently.
- Create a stronger focal point.
- Remove unnecessary elements.
- Increase whitespace.

---

## 10.2 Typography

Typography checks font choice and text presentation.

The AI should review:

- Font selection
- Font pairing
- Font size
- Font weight
- Line height
- Letter spacing
- Text alignment
- Type hierarchy
- Tone match
- Readability

Common issues:

- Font does not match the brand.
- Too many fonts are used.
- Text is too small.
- Headline and body text look similar.
- Font feels childish, outdated, or mismatched.
- Line spacing is uncomfortable.

Recommendations may include:

- Use fewer fonts.
- Create a clear type scale.
- Increase headline contrast.
- Use stronger font pairing.
- Choose fonts matching the audience and tone.
- Consider fonts such as Inter, Poppins, Montserrat, DM Sans, Manrope, Playfair Display, or Lora depending on the design category and brand tone.

---

## 10.3 Color Theory

Color theory checks whether the color palette supports the message, mood, and audience.

The AI should review:

- Color harmony
- Contrast
- Brand fit
- Emotional tone
- Color consistency
- Accessibility contrast
- Background and foreground readability
- Overuse of colors

Common issues:

- Colors do not match the target audience.
- Low contrast makes text hard to read.
- Too many colors create visual noise.
- Colors feel generic or outdated.
- CTA color does not stand out.
- Serious brand uses overly playful colors.

Recommendations may include:

- Reduce palette to 2–4 main colors.
- Use one accent color for actions.
- Improve contrast.
- Choose colors based on industry expectations.
- Use accessible color combinations.
- Create a primary, secondary, neutral, and accent palette.

---

## 10.4 Spacing

Spacing checks the distance between elements.

The AI should review:

- Margins
- Padding
- Gaps between text and images
- Section spacing
- Grouping
- Breathing room
- Consistency

Common issues:

- Elements are too close together.
- Spacing feels random.
- Important elements do not have enough breathing room.
- Sections are not clearly separated.
- Content feels cramped.

Recommendations may include:

- Use consistent spacing units.
- Increase whitespace around important elements.
- Group related items closer together.
- Separate unrelated sections.
- Use spacing scale such as 4, 8, 16, 24, 32, 48, 64.

---

## 10.5 Hierarchy

Hierarchy checks whether the most important information is seen first.

The AI should review:

- Main focal point
- Headline importance
- CTA visibility
- Content order
- Size contrast
- Color contrast
- Weight contrast
- Visual priority

Common issues:

- Everything looks equally important.
- CTA is weak.
- Headline does not stand out.
- Secondary content dominates.
- The user's eye does not know where to go first.

Recommendations may include:

- Make the headline larger or bolder.
- Reduce visual weight of secondary text.
- Use size, spacing, contrast, and placement to guide attention.
- Make CTA more visible.
- Use one clear primary message.

---

## 10.6 Readability

Readability checks how easily people can read and understand the design.

The AI should review:

- Text size
- Contrast
- Sentence length
- Background distractions
- Line spacing
- Text placement
- Visual noise
- Scanability

Common issues:

- Text is too small.
- Background makes text hard to read.
- Too much text in one area.
- Low contrast.
- Poor line height.
- Text does not scan well.

Recommendations may include:

- Increase font size.
- Improve contrast.
- Break text into smaller sections.
- Use bullets or shorter copy.
- Add background overlay behind text.
- Simplify content.

---

## 10.7 Creativity

Creativity checks whether the design feels original, memorable, and visually interesting.

The AI should review:

- Originality
- Concept strength
- Visual metaphor
- Style uniqueness
- Memorability
- Emotional impact
- Avoidance of generic templates

Common issues:

- Design looks like a template.
- No unique visual idea.
- Too predictable.
- No memorable element.
- Style does not support the message.

Recommendations may include:

- Add a stronger concept.
- Use a unique visual hook.
- Simplify and strengthen the main idea.
- Avoid unnecessary decoration.
- Make the design more ownable to the brand.

---

## 10.8 Audience Fit

Audience fit checks whether the design matches the intended viewers.

The AI should review:

- Age group match
- Industry match
- Cultural tone
- Brand personality
- User expectations
- Emotional response
- Professional level

Common issues:

- Design is too playful for a serious audience.
- Design is too corporate for a young audience.
- Visual tone does not match the product.
- Typography or colors send the wrong message.
- Design does not build trust.

Recommendations may include:

- Adjust color tone.
- Choose more appropriate fonts.
- Change imagery style.
- Simplify or energize layout depending on audience.
- Align the visual language with user expectations.

---

## 10.9 Accessibility

Accessibility checks whether more people can use and understand the design.

The AI should review:

- Color contrast
- Text readability
- Font size
- Visual clarity
- Touch target size for UI
- Alt text suggestions
- Color-only communication
- Layout clarity

Common issues:

- Low contrast text.
- Important information shown only through color.
- Tiny buttons.
- Dense text.
- Hard-to-read decorative fonts.
- UI elements too close together.

Recommendations may include:

- Improve contrast ratio.
- Use clearer font sizes.
- Add labels along with color indicators.
- Increase button size.
- Avoid overly decorative type for important information.
- Check accessibility standards for UI designs.

## 11. Feedback Structure

Every review should follow a consistent structure.

### 11.1 Overall Summary

The AI gives a quick summary of the design.

Example:

```text
The design has a strong visual direction, but the typography and spacing weaken its professional feel. The main issue is that the hierarchy is unclear, so the viewer does not immediately understand what to focus on.
```

---

### 11.2 Overall Score

Example:

```text
Overall Design Score: 6.8/10
```

The score should not stand alone. It must be explained.

---

### 11.3 Category Scores

Example:

```text
Composition: 7/10
Typography: 5/10
Color Theory: 6/10
Spacing: 6/10
Hierarchy: 5/10
Readability: 7/10
Creativity: 6/10
Audience Fit: 5/10
Accessibility: 6/10
Industry Fit: 7/10
```

---

### 11.4 What Is Working

The AI should identify strengths before moving into critique.

Examples:

- The layout has a clean starting structure.
- The color palette has potential.
- The main message is understandable.
- The design has a good visual direction.
- The imagery supports the concept.
- The CTA is present and visible.

---

### 11.5 Issues and Improvements

Each issue should include:

- What is wrong.
- Why it is wrong.
- How to improve it.
- Specific recommendations.

Required structure:

```text
Category: Typography
Score: 5/10

What is wrong:
The chosen font does not match the target audience.

Why:
The font creates a playful feeling while the design aims for a serious tone.

How to improve:
Use a more neutral and professional typeface with better readability.

Specific recommendations:
Consider fonts such as Inter, Poppins, or Montserrat.
Increase headline weight.
Reduce body text size slightly.
Improve line spacing.
```

---

### 11.6 Priority Level

Each issue should have a priority.

Priority levels:

- High priority
- Medium priority
- Low priority

Example:

```text
Priority: High

Reason:
This issue directly affects readability and first impression.
```

---

### 11.7 Action Checklist

The AI should end with a clear improvement checklist.

Example:

```markdown
### Fix Checklist

- Increase headline size by 20–30%.
- Use one primary font family.
- Improve text/background contrast.
- Reduce the number of colors.
- Add more spacing around the CTA.
- Align all left-side text to the same grid.
- Make the CTA more visually dominant.
```

## 15. AI System Logic

## 15.1 Inputs Required by AI

The AI should consider:

- Uploaded design image
- Design category
- Target audience
- Purpose
- Style
- Goals
- Feedback mode
- Industry
- Platform
- User concern
- Previous user questions if available

---

## 15.2 AI Review Process

The AI should follow this internal review process:

1. Understand the design category.
2. Understand the design brief.
3. Identify the main visual goal.
4. Analyze the design visually.
5. Check category-specific principles.
6. Score review categories.
7. Identify strengths.
8. Identify weaknesses.
9. Explain why each issue matters.
10. Recommend fixes.
11. Prioritize improvements.
12. Suggest follow-up actions.
13. Offer optional improved version.

---

## 15.3 AI Output Rules

The AI should avoid:

- Vague feedback.
- Generic praise.
- Insulting language.
- Overly technical explanations in Beginner Mode.
- Redesigning without user permission.
- Ignoring the design brief.
- Judging only by aesthetics.
- Giving scores without explanation.

The AI should always provide:

- Clear reasoning.
- Practical fixes.
- Category-specific analysis.
- Specific recommendations.
- Priority order.
- Educational value.

# 19. AI Review Output Example

## Example Review

```markdown
## Overall Score: 6.2/10

The design has a good starting concept, but the visual hierarchy and typography need improvement. The main message is not immediately clear, and the current font choice does not fully match the serious tone described in the brief.

### What Works

- The layout has a clean base.
- The color palette has potential.
- The main subject is visible.
- The design does not feel overly cluttered.

### Typography: 5/10

What is wrong:
The chosen font does not match the target audience.

Why:
The font creates a playful feeling while the design aims for a serious tone.

How to improve:
Use a cleaner, more professional sans-serif font.

Specific recommendations:
- Try Inter, Poppins, or Montserrat.
- Increase headline weight.
- Improve line spacing.
- Reduce decorative styling.

Priority:
High

### Final Fix Checklist

- Improve headline hierarchy.
- Use a more suitable font.
- Increase spacing around the main content.
- Improve contrast between text and background.
- Make the CTA more visible.
- Reduce unnecessary visual elements.
```

# 21. Things to Avoid

The product should avoid:

- Only giving generic scores.
- Saying “looks good” without explanation.
- Being only a design generator.
- Ignoring the user’s brief.
- Treating all design categories the same.
- Making Brutally Honest Mode rude or personal.
- Overcomplicating MVP with community too early.
- Adding too many advanced features before the core review experience is strong.
- Giving font or color suggestions without explaining why.
- Creating feedback that is too vague to act on.

# 22. Final Product Direction

AI Design Mentor should begin as a focused AI critique platform.

The first version should solve one problem extremely well:

**A user uploads a design and receives clear, professional, actionable feedback that explains what is wrong, why it is wrong, and how to improve it.**

After this core experience becomes strong, the platform can expand into:

- AI improvements
- Before/after comparison
- Progress tracking
- Design challenges
- Community critiques
- Designer profiles
- Portfolio building
- Team and freelancer tools

The long-term vision is to become a complete design improvement platform where users can learn, improve, compare, share, and build a stronger portfolio through structured AI and community feedback.
